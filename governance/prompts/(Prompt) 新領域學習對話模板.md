# 新領域學習對話模板

> 以下模板可直接複製貼到 Claude / ChatGPT 使用。AI 看到這類格式也應主動識別並套用對應執行流程。

---

## Prompt A：知識地圖建構（Phase 1 入口，必做）

```
你是一個專業的 [領域名稱] 課程設計師，擁有 10 年以上的教學和實戰經驗。

我是一個完全零基礎的新人，背景是 [使用者現有背景]。

請幫我做兩件事：

1. 畫出這個領域的完整知識地圖（MECE 結構），按照學習依賴順序排列每個知識節點
2. 特別標記：哪些是「大多數教材不會主動提到，但業界默認你已經知道」的隱性知識

格式：樹狀條列，每個節點加上「初學者常見誤解」備註。
```

---

## Prompt B：遞迴前置知識挖掘（挖掘 Unknown Unknowns）

```
我想學 [具體技能或概念]。

請用遞迴的方式，告訴我：
- 在學這個之前，我需要先知道什麼？
- 那個前置知識之前，又需要什麼？
- 一直往前推，直到找到「任何正常人都已知道」的起點

同時標記：哪些前置知識是「大多數人跳過後才後悔的」？
```

---

## Prompt C：多角色矛盾訪談（找出判斷點）

```
請分別扮演以下三個在 [領域名稱] 有豐富經驗、但背景和立場不同的行業老手：

角色一：[立場 A，例如：注重實用效率的從業者]
角色二：[立場 B，例如：注重學術理論的專家]
角色三：[立場 C，例如：注重商業結果的實踐者]

針對問題：[具體問題或選擇]

請讓三個角色各自表達立場，允許他們互相矛盾。
矛盾的地方不需要給出「正確答案」，而是說明為什麼不同立場會產生不同判斷。
```

---

## Prompt D：跨域類比（加速建立直覺）

```
我有 [已知領域，例如：程式開發 / 美術設計] 的背景。

請幫我把 [新領域] 的以下概念，對應到我熟悉的 [已知領域] 概念：
[列出 5-10 個新領域的核心概念]

對每個類比：
- 說明類比「合」的地方（可以直接遷移的直覺）
- 說明類比「不合」的地方（這個領域特有的邏輯，需要重新建立）
```

---

## Prompt E：認知誤區掃描（進入子領域前的掃雷）

```
針對 [領域] 的 [子領域] 這個主題：

請列出：
1. 初學者最常有的 3 個認知誤區
2. 每個誤區通常在什麼階段才會被發現
3. 發現後的正確理解是什麼

不要給我「正確知識」，給我「哪裡最容易錯，以及為什麼」。
```

---

## Prompt F：費曼測試（杨竹筠 MIT 學習法核心步驟）

```
我剛才學習了 [主題 / 概念]，以下是我的理解：

[使用者用自己的話說明理解]

請以原始學習資料為準，告訴我：
1. 我的理解有哪些是正確的？
2. 哪些地方有誤解或遺漏？
3. 哪些核心邏輯我可能只是記住了表面，但沒有真正理解？
```

---

## Prompt G：壓力測試（杨竹筠 MIT 學習法核心步驟）

```
針對 [主題 / 概念]，請提出 10 個問題，用來測試一個人是否真正理解這個主題，
而不只是死記硬背了一些事實。

問題應該：
- 包含需要舉例說明的問題
- 包含需要說明「為什麼」的問題
- 包含需要比較兩個概念差異的問題
- 包含需要應用在新情境的問題

（我會在回覆中嘗試回答這些問題，請之後根據原始資料給我回饋）
```

---

## 使用說明

| Prompt | 使用時機 | 必做/選用 |
|--------|---------|---------|
| A | 進入任何新領域的第一步 | 必做 |
| B | 針對 A 地圖中感興趣的節點深挖 | 必做 |
| C | 遇到有爭議的選擇或判斷點時 | 選用 |
| D | 使用者有跨域背景時 | 選用 |
| E | 進入每個子領域實作前 | 選用 |
| F | 每次學習後，驗證自己的理解 | 選用（MIT 法推薦必做） |
| G | 需要壓力測試自己的理解時 | 選用（MIT 法推薦必做） |
