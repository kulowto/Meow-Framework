# (Prompt) 組織圖與系統架構圖輸出規範

## 輸出格式規定

當需要生成任何**組織圖、系統架構圖、流程圖、層級關係圖**時，
一律使用 **Obsidian Canvas JSON 格式**輸出，副檔名為 `.canvas`。

---

## 格式規範

### 基本結構

```json
{
  "nodes": [ ... ],
  "edges": [ ... ]
}
```

### 節點（nodes）欄位

| 欄位 | 說明 |
|------|------|
| `id` | 唯一識別碼，英文命名，不可重複 |
| `type` | `text`（一般節點）或 `group`（分組容器） |
| `text` | 節點顯示文字（group 類型用 `label`） |
| `x` / `y` | 節點座標位置 |
| `width` / `height` | 節點尺寸 |
| `color` | 顏色編號（見下方色碼對照） |

### 連線（edges）欄位

| 欄位 | 說明 |
|------|------|
| `id` | 唯一識別碼 |
| `fromNode` | 起點節點 id |
| `fromSide` | 起點方向：`top` / `bottom` / `left` / `right` |
| `toNode` | 終點節點 id |
| `toSide` | 終點方向：`top` / `bottom` / `left` / `right` |
| `label` | 連線上的說明文字（選填） |
| `color` | 連線顏色編號（選填） |

### 色碼對照

可用顏色編號為 `1` 至 `6`，對應如下：

| 編號 | 顏色 |
|------|------|
| `1` | 紅色 |
| `2` | 橙色 |
| `3` | 黃色 |
| `4` | 綠色 |
| `5` | 藍紫色 |
| `6` | 紫色 |

依照當前圖的內容與節點類型，自行評估分類邏輯並分配顏色，確保同類型節點使用一致的顏色編號。

---

## 排版原則

- 由上至下排列為主（`y` 值遞增）
- 同層級的節點對齊同一 `y` 值
- 分組容器（`group`）的座標要能包住內部所有節點，留 20px 以上的邊距
- 節點預設寬度建議 `160–220`，高度建議 `60–80`
- 節點間距建議至少 `20px`

---

## 注意事項

- **不使用 Mermaid**，不輸出 SVG，不輸出 PNG
- 所有圖一律輸出為 `.canvas` 檔案
- 使用者拿到後可在 Obsidian Canvas 中自由拖拉調整位置
- 若架構有「現階段 vs 未來」的差異，未來規劃的節點用顏色 `6`（紫色）標示
- 回饋迴路（如錯誤回饋）用虛線或加上 `label` 說明方向

---

## 使用範例

> 「請幫我畫一張 XX 系統的架構圖」
> → 直接輸出 `.canvas` 檔，不問格式，不用 Mermaid
