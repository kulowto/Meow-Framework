# 文案領域文件

## 文件說明

- 適用角色：文案撰寫、文案 QA、文案優化
- 繼承自：全局文件（語言風格、回覆邏輯、資料傳遞規則均沿用，不重複定義）
- 本文件定義文案領域的撰寫規範，主要應用場景為**影片腳本**與**社群文案**
- 語氣規範（Voice / Tone）是文案領域最核心、也最需要人工校準的部分，目前為草稿版本

---

## §1 基礎規範

### 1.1 Voice（固定人格特質）

> Voice 是頻道/品牌永遠不變的人格特質，不因內容主題改變。
> 注意：Voice 固定，Tone 隨情境切換（見 §3.2）。
>
> ⚠️ 以下三個特質為草稿版本，使用者確認後定稿。
> Voice 特質一旦定稿，後續只能擴充說明，不能輕易改動（因為所有文案角色都繼承）。

#### 直白（Direct）

說重點，用最短的路徑讓觀眾理解，不繞彎、不鋪陳。

| ✅ 這樣說 | ❌ 不這樣說 |
|---------|-----------|
| 「這個方法有效，因為它解決了 X 問題。」 | 「透過本次的分享，希望能讓大家了解這個方法的重要性。」 |
| 「結論是：先做 A，再做 B。」 | 「綜合以上所述，我們可以得出以下結論…」 |
| 「這不適合初學者。」 | 「可能對於還在學習階段的朋友，相對會有一些門檻。」 |

#### 真實（Genuine）

說自己真正相信的，不誇大效果，不說教，分享真實觀察與親身經歷。

| ✅ 這樣說 | ❌ 不這樣說 |
|---------|-----------|
| 「我當時也犯過這個錯誤，後來才發現…」 | 「相信很多人都曾經歷過…（實際上自己沒有）」 |
| 「這方法對我有效，但不保證對你一樣。」 | 「只要照這樣做，你一定能成功！」 |
| 「坦白說，我覺得這個工具沒那麼好用。」 | 「這個工具絕對是目前市面上最好的選擇！」 |

#### 有溫度（Warm）

知道螢幕另一端是活生生的人，在適當情境下加入感性描述與人情味。不是每句話都要感性，而是在對的地方讓觀眾感受到「你懂我」。

| ✅ 這樣說 | ❌ 不這樣說 |
|---------|-----------|
| 「那段時間真的很難熬，我記得…」 | 「那段時間有一些困難，後來我克服了它。」 |
| 「如果你也有這種感受，你不是一個人。」 | （沒有這句話，直接跳到解方）|
| 「慢慢來，這件事急不得。」 | 「請按照步驟逐一執行即可。」 |

---

### 1.2 詞彙規範

**✅ 應該使用**：
- 口語化用語（說話的感覺，不是寫報告的感覺）
- 主動語態（「我做了 X」而非「X 被做了」）
- 短句優先（一句一個意思，遇到複合句就拆開）
- 具體數字和例子（「少 3 個步驟」而非「更簡單」）

**❌ 不應該使用**：
- 書面連接詞：「旨在」、「藉此」、「然而」、「因此」、「綜上所述」、「透過…的方式」
- 誇大形容詞：「最好的」、「革命性的」、「顛覆性的」（除非有具體依據）
- 說教語氣：「大家應該要…」、「必須明白…」、「這是不對的行為」
- 模糊承諾：「你一定能成功」、「保證有效」、「百分之百可行」

---

### 1.3 內容哲學

這三條原則影響所有類型的文案，不分模板或情境：

- **情境代入 > 純乾貨**：知識要用「具體場景 + 解法」呈現，直接說理論只吸引同行，吸引不到目標受眾
  - ❌ 「PVP 的核心是資源控制。」
  - ✅ 「你有沒有遇過：明明打贏每場單打，最後卻輸了？那通常是資源控制沒做好。」
- **用娛樂的方式傳播專業**：讓觀眾在有趣的過程中獲得有用的知識，而不是坐在教室裡聽課
- **資訊密度 > 影片長度**：每一秒都有料，才有人看完；不是因為短，而是因為濃縮

## §2 結構規範（腳本模板）

> 三個模板對應三種最常見的內容類型。
> 角色 SOP 宣告「這個角色預設使用哪個模板」，不在 SOP 裡重複定義模板骨架。

### 通用骨架（所有模板適用的宏觀結構）

每集影片無論使用哪個模板，都遵循此宏觀結構：

```
開頭（前 5-10 秒）→ 開場總結（一句話說這集的目的）→ 主體內容 → 收尾
```

**開場總結**：接在開頭 Hook 之後，用一句話讓觀眾知道「這集會得到什麼」。  
讓沒興趣的人早點離開，有興趣的人安心繼續看。  
例：「這集幫你解析三個 PVP 常見陣容，找到適合自己階段的選擇。」

---

### 模板 A：Hook-Body-CTA（教學 / 資訊型）

- **適用場景**：教學、工具測評、資訊分享、快節奏主題、以「給出答案」為目的的內容
- **Tone 對應**：「直白（Direct）」為主，可依內容適當帶入「有溫度（Warm）」

**骨架**：

```
[Hook] 0–5 秒
  一句話挑起好奇或直接說出結果，停住觀眾繼續看
  例：「我花了三年才學會這件事，你花三分鐘就能懂。」

[Body] 依影片長度
  兌現 Hook 的承諾，直接給內容
  每個段落只說一件事
  轉場用短句引導，例：「第二件事是…」、「接著來看…」

  ▸ 教學型 Body 的展開格式（每個觀念依序展開）：
    - 標題：一句話總結這個觀念
    - 誤區（可選）：先說常見錯誤，更能抓注意力
    - 說明：解釋原因或邏輯
    - 舉例：用生活案例類比，讓不熟悉的觀眾也能理解

[CTA] 最後 1–3 秒
  明確指令，告訴觀眾下一步是什麼
  例：「喜歡這類內容的話，訂閱讓我知道。」
```

**禁止事項**：
- Hook 超過 10 秒才切入重點
- CTA 模糊（「可以考慮看看」、「有興趣的話再…」）
- Body 同時塞超過三個重點

---

### 模板 B：PAS（個人故事 / 問題導向型）

- **適用場景**：個人過往經歷、情感共鳴型內容、痛點描述、說服觀眾理解某個觀點
- **Tone 對應**：Problem 和 Agitate 段用「有溫度（Warm）」，Solution 段切換「直白（Direct）」

**骨架**：

```
[Problem] 點出問題
  描述觀眾可能正在面對的情境或痛點
  例：「你有沒有那種，明明努力了很久，還是沒有進展的感覺？」

[Agitate] 放大感受
  帶入情境，讓觀眾感受到「你懂我」
  不是誇大問題，是誠實地說出那種感受，不跳過這一步
  例：「我那時候每天都在懷疑自己，到底哪裡出問題了…」

[Solution] 提供解方
  清楚說明解法，恢復直白語氣
  例：「後來我發現，問題不是努力不夠，而是方向。這是我怎麼調整的…」
```

**禁止事項**：
- 跳過 Agitate 直接給 Solution（失去情感連結，變成說教）
- Agitate 比例超過整體 30%（觀眾會不耐煩）
- Solution 段繼續用感性語氣（會讓解方看起來不具體）

---

### 模板 C：AIDA（促購 / 行動引導型）

- **適用場景**：推廣內容、號召訂閱、產品或課程推薦、行動引導
- **Tone 對應**：有力、帶急迫感，保持「真實（Genuine）」，不說過頭

**骨架**：

```
[Attention] 吸睛開場
  一個問題、數字或反直覺說法，停住觀眾
  例：「90% 的人都做錯了這件事。」

[Interest] 建立興趣
  解釋為什麼這對觀眾重要，讓他們繼續看
  例：「這個錯誤，讓你白花了三倍的時間。」

[Desire] 激起欲望
  描述「有了這個之後的樣子」，具體且真實
  例：「掌握這個方法之後，我每週省下 5 小時。」

[Action] 明確行動
  清楚告訴觀眾下一步，越具體越好
  例：「連結在下方，今天就去試試。」
```

**禁止事項**：
- Attention 使用誇張說法但 Body 沒有兌現（誘導式，傷信任）
- Action 沒有具體指令（「大家可以參考看看」無效）
- Desire 使用「一定成功」、「保證有效」等誇大承諾

## §3 執行方法論

### 3.1 核心框架

**文案三角**：每次撰寫前，先確認這三個要素：
1. **受眾**：這段內容是說給誰聽的？他們的背景和痛點是什麼？
2. **目的**：這段話想讓觀眾「做什麼」或「感受到什麼」？
3. **核心訊息**：如果只能說一件事，是哪一件？

三個要素不清楚，選模板和定語氣都會跑偏。

---

### 3.2 Tone 切換規則

> Voice 固定不變，Tone 隨情境調整。
> 判斷方式：先確認「這段內容的主要目的」，再對照下表選 Tone 設定。

| 情境類型 | 觀眾當下狀態 | Tone 設定 | 具體調整方式 |
|---------|------------|----------|------------|
| 教學 / 資訊型 | 想學東西，期待答案 | 清楚、直接 | 短句、條列、不廢話 |
| 個人故事 / 過往經歷 | 可能有共鳴，正在決定要不要繼續看 | 溫暖、有共鳴 | 帶感性細節、可用疑問句引共鳴、保留人情味 |
| 促購 / 行動引導 | 考慮中，需要一個理由 | 有力、帶急迫感 | 動詞開頭、明確指令、避免模糊 |
| 娛樂 / 輕鬆話題 | 放鬆瀏覽，低專注 | 幽默、隨性 | 口語化、可帶梗，但梗要自然，不強迫 |
| 觀點分享 / 問題探討 | 思考中，想聽立場 | 中性、有觀點 | 說清楚立場，但不說教 |

**跨段切換注意**：同一支影片若不同段落需要切換 Tone，要有過渡句，不能突然切換。例如從感性的 Agitate 切換到直白的 Solution 時，可以用：「但其實，解法沒有你想的複雜。」

---

### 3.3 Hook 設計方法

Hook 是觀眾決定「要不要繼續看」的關鍵 3-5 秒。

| Hook 類型（別名） | 例子 | 最適合搭配的模板 |
|---------|------|-------------|
| 說出結果（重點型） | 「我用這個方法，三個月改掉了一個壞習慣。」 | 模板 A |
| 反直覺問題（觀點型） | 「你以為努力就夠了嗎？」 | 模板 B |
| 數字衝擊（破題型） | 「90% 的人都不知道這件事。」 | 模板 C |
| 代入痛點（反問型） | 「有沒有那種明明努力卻沒進展的感覺？」 | 模板 B |
| 預告結果（破題型） | 「看完這支影片，你會知道為什麼你一直失敗。」 | 模板 A / C |

---

### 3.4 自我檢查方法

1. **大聲朗讀**：完成後大聲朗讀一次，讀起來卡口的地方就是要改的地方
2. **書面語掃描**：全文搜尋「旨在」、「藉此」、「然而」、「綜上」等關鍵詞
3. **目的檢查**：最後一段能不能清楚說出「觀眾看完這段，我希望他做什麼或感受到什麼」

---

### 3.5 受眾設計流程

在規劃任何系列或單集影片時，依序確認三個問題：

1. **主受眾是誰**：這批內容主要是對誰說的？他們的背景、痛點、目標是什麼？
2. **延伸受眾有哪些**：這群主受眾還對什麼議題感興趣？這些延伸議題可以做成額外的影片，吸引更廣的觀眾
3. **如何引導回主題**：延伸內容做完之後，如何讓新進觀眾對主線系列產生興趣？

**設計邏輯**：不要只鎖定最核心的受眾，純專業內容只吸引同行。先從延伸議題觸及更廣的受眾，再引導他們進入主線。

---

### 3.6 系列設計原則

製作連載系列時，每集影片必須同時滿足兩個條件：

- **系列感**：看完這集之後，觀眾會想看下一集（有期待感）
- **獨立成集**：新觀眾第一次看這集，不需要看前幾集也能有收穫，不會因為沒背景就聽不懂

兩個條件不能只顧一個。只顧系列感 → 新觀眾進不來。只顧獨立完整 → 老觀眾沒有黏著感。

## §5 品質 Checklist

- [ ] Hook 在前 5 秒能引起好奇或共鳴（對照「Hook 設計方法」）
- [ ] Hook 後有一句「開場總結」說明這集的目的
- [ ] 全篇沒有書面語（搜尋：旨在、藉此、然而、綜上所述）
- [ ] 每個段落只說一件事
- [ ] 知識是用「具體情境 + 解法」呈現，而非純乾貨說理
- [ ] Tone 符合內容類型（對照 §3.2 情境對應表）
- [ ] CTA 明確且可執行（不是模糊指令）
- [ ] 大聲朗讀一次，確認沒有卡口的句子
- [ ] 使用的腳本模板與內容類型吻合（A / B / C）
- [ ] 沒有誇大承諾（「一定成功」、「保證有效」）
- [ ] 若為系列集數：本集能獨立成立，新觀眾不需看前集也能有收穫
